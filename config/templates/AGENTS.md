# CTF pwn workspace

This directory is a CTF pwn challenge workspace generated by `pwnnew`.

## Commands

- Local run: `python3 solve.py`
- GDB run: `python3 solve.py GDB`
- Remote run: `python3 solve.py REMOTE HOST=<host> PORT=<port>`
- Select binary: `python3 solve.py BIN=./chall`
- Select libc/ld: `python3 solve.py LIBC=./libc.so.6 LD=./ld-linux-x86-64.so.2`

## Workflow preferences

- Keep original challenge files unchanged. Copy before patching.
- Prefer pwntools for scripting and pwndbg for debugging.
- Use `checksec`, `file`, `ldd`, `patchelf`, `ROPgadget`, `ropper`, `one_gadget`, `seccomp-tools`, `libc-database`, and `glibc-all-in-one` when useful.
- Keep exploit notes inside this workspace.
- Do not commit large downloaded libc databases or extracted tool caches unless explicitly needed.

## Debugging notes

- If a custom loader is provided, run with `LD=./ld-linux... LIBC=./libc.so.6`.
- If remote behavior differs, check buffering, environment, PIE/ASLR, and libc version.
- For heap challenges, record glibc version and allocator assumptions before writing the final exploit.
